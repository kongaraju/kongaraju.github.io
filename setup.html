Limitations

All limitations mentioned in this section will be fixed in coming releases.
  • Client cannot run in multiple browser tabs.
  • Client runs only in google chrome browser.
  • Server cannot scale to muliltple physical/virtual machines.
  • Editing or viewing of office documents is not supported for now.
  • Only debian packaging is supported for now.

Requirements

Client : Client requires google chrome browser (version > 29).
Server : 64 bit ubuntu ( > 11 ) is required.

Installation

Package can be installed with the below command
sudo gdebi antkorp_1.0_ubuntu_limited_alpha_amd64.deb

We strongly recommend you to install this software on a non production server. This is an alpha quality software.

Server uses self signed certificates for SSL/TLS authentication, replace these with that of your organization if you want to access the server on the internet.

By default client uses google public stun server for webrtc signalling, replace this with your own.
STUN/TURN server address can be changed from antkorp configuration utility.

Uninstallation

Package can be removed with the below command
sudo apt-get remove --auto-remove antkorp;

How to access application

After successful installation application can be accessed with
https:// < ip_address >/antkorp

NOTE: Antkorp uses 8080 as default port for communication between client and server. Please make sure this port is open in the firewall for the application to work successfully. Default port can be changed at any time using antkorp configuration dialog.

Administration

Antkorp platform can be started with command
start antkorp

restart antkorp with
restart antkorp

stop antkorp with
stop antkorp

configure antkorp server with
/opt/antkorp/custom/bin/configui.sh

Debugging:

Unable to access the application:
  1. Firewall issue: Please check if the firewall port (8080) is open.
    firewall port can be opened by command
    sudo iptables -A INPUT -p tcp --dport 8080 -j ACCEPT

  2. Please use "https" instead of "http" .

  3. Enabling https on server
    The following commands will enable the https on the webserver.
    sudo a2enmod ssl
    sudo a2ensite default-ssl


    Open the ssl configuration file and change the location of the certificate and key files.
    sudo vi /etc/apache2/sites-available/default-ssl.conf

    Locate the variables "SSLCertificateFile" and "SSLCertificateKeyFile" comment out the variables and paste the below 2 lines.

    SSLCertificateFile /opt/antkorp/certs/selfsigned.pem
    SSLCertificateKeyFile /opt/antkorp/certs/selfsigned.key

    restart the apache server as below
    sudo service apache2 restart

    if the restart fails change the permissions of the key files with
    sudo chown root:root /opt/antkorp/certs/selfsigned.pem
    sudo chown root:root /opt/antkorp/certs/selfsigned.key

    again restart the server with
    sudo service apache2 restart
